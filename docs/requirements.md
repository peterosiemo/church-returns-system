
# Church Returns System Requirements

## Purpose

The system will manage church returns submitted by churches within different regions.

## Monthly Returns

Each church submits:

1. Tithes
2. Member Contribution

## Yearly Contributions

Each church contributes:

1. Men Conference
2. Women Conference
3. Youth Conference
4. Main Conference

## Reporting

The system should provide:

- Church reports
- Regional reports
- National reports
- Monthly trends
- Yearly trends
- Missing return alerts

## User Roles

- Administrator
- Regional Overseer
- Data Entry Clerk

## Analytics

- Best performing church
- Best performing region
- Growth trends
- Missing returns detection
